# v0.8.0 — Compliance & Hardening

**Status:** Implemented 2026-05-17
**Issue:** [#38](https://github.com/Nolpak14/agorio/issues/38)
**Target buyer:** Procurement / legal / IT teams evaluating an agent platform ahead of the
2 Aug 2026 EU AI Act deadline.

## Why this release

The Aug 2026 EU AI Act deadline is a hard forcing function for buyer demand. Shipping the
compliance posture in Q4 2026 means agorio is procurement-ready before the typical enterprise
budget cycle for 2027. Concretely, every enterprise RFP we've seen has asked for:

1. An exportable audit trail (Annex IV) — the **compliance export endpoint** answers this.
2. Proof that the agent is who it claims to be — the **`AgentAttestation`** primitive answers this.
3. A published security posture — **`docs/security.md`** + npm audit + secret scanning answer this.
4. Multi-platform coverage — the **BigCommerce adapter** brings us to three real merchants.

## Scope

### SDK

- ✅ BigCommerce adapter with feature parity to Shopify/WooCommerce (`src/adapters/bigcommerce.ts`)
- ✅ AP2 client GA — deprecate `experimental_ap2`, promote to `ap2` (kept dual-honored through v0.x)
- ✅ `verifyMandateShape()` receiver helper for AP2 merchants
- ✅ `AgentAttestation` HMAC-based agent identity (`src/security/agent-attestation.ts`)

### Cloud

- ✅ EU AI Act compliance export endpoint + dashboard page
- ✅ `cloud_audit_log` table + `appendAudit()` helper wired into existing actions
- ✅ `/audit-log` viewer page
- ✅ RBAC schema (`orgs`, `org_members`, `org_role` enum) — migration only; UI gating is a v1.0 follow-up

### Docs

- ✅ `docs/security.md` (OWASP + dependencies + vuln disclosure)
- ✅ `docs/compliance.md` (EU AI Act, GDPR, PCI, SOC 2, HIPAA, ISO 27001)

### Bench

- ✅ `bench/run.ts` harness covering UCP discovery, product search, single-tool-call, full
  checkout. Stub LLM keeps results provider-independent.

## Deferred to v1.0

- Helm chart (the Docker Compose bundle in `docker/` is good enough for v0.8 self-host)
- Third-party penetration test report (engaging vendor in Q4 2026)
- Argon2id hashing of API keys at rest
- Right-to-erasure endpoint (today: operator-assisted)

## Cut from scope

- BotID integration — chose HMAC attestation as a strictly simpler primitive that requires
  zero third-party signup. See [`docs/adr/0006-attestation-hmac.md`](../adr/0006-attestation-hmac.md).

## Migration

- `experimental_ap2` is still honored through the v0.x line. Update at your leisure; the
  removal lands in v1.0 per the deprecation policy.
- Customers running self-hosted Cloud should run `npm run db:push` after pulling to apply the
  new `orgs`, `org_members`, and `cloud_audit_log` tables.
