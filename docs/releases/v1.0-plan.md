# v1.0.0 — Production GA

**Status:** Planning (foundations landed alongside v0.8)
**Issue:** [#39](https://github.com/Nolpak14/agorio/issues/39)
**Target:** H1 2027 — aligned with enterprise 2027 budget cycles.

## Goal

Declare agorio production-ready. Stability guarantees, semantic versioning commitment, full
protocol coverage, enterprise SSO, and a comprehensive documentation site.

## Foundations landed in v0.8

These v1.0 items shipped alongside the compliance work because they were ready:

- ✅ `cloud_audit_log` table + viewer page
- ✅ RBAC schema (`orgs`, `org_members`) — migration only; role enforcement is the v1.0 work
- ✅ Performance benchmark harness (`bench/run.ts` + `baseline-v0.8.0.json`)
- ✅ Semantic versioning policy (`docs/semver.md`)
- ✅ Migration guide 0.x → 1.0 (`docs/migration-0.x-to-1.0.md`)
- ✅ ADR series (`docs/adr/0001-0007.md`)
- ✅ Self-hosted Docker Compose (`docker/docker-compose.yml`, `docker/cloud.Dockerfile`)
- ✅ Adapter SDK guide + community plugin program docs
- ✅ Plugins / adapters registries

## Definition of done

Lifted directly from issue #39:

- v1.0.0 published to npm with no breaking changes for 90 days after release
- Public status page shows >99.9% Cloud uptime for the prior 90 days
- At least 3 enterprise customers running production agents on Cloud with SSO
- Docs site has full guides + API reference, no "TODO" sections

## Strategic rationale

v1.0 is the maturity signal that enterprise procurement teams require before adopting
infrastructure for long-running production agents. The v0.8 compliance posture clears the
legal hurdle; v1.0 clears the technical-risk hurdle.
