# Agorio v0.5 ‚Äî Open Core Release Plan

> **Status: Shipped ‚Äî v0.5.0, May 2026.** This document is now a historical artifact. See [`docs/ROADMAP.md`](ROADMAP.md) for the current plan.
>
> **Original target:** late May / early June 2026 (~3 weeks of work)
> **Theme:** Resolve the plugin paywall ambiguity, ship AP2 + WooCommerce + Shopify-UCP-migration support.

## Context

v0.4.2 shipped a monetization scaffold (Stripe + Neon + customer dashboard + 5 plugins with `AGORIO_LICENSE_KEY` checks) but the plugins were never published to npm and the license check is a misleading `console.warn`. Meanwhile, the agentic-commerce landscape shifted:

- **Shopify migrated its Storefront MCP server to UCP on Apr 22, 2026** ‚Äî effective date May 30, 2026. Every Shopify store will be a UCP merchant at `/.well-known/ucp`. Agorio's `UcpClient` must handle the new profile shape.
- **AP2 (Agent Payments Protocol) moved to FIDO Alliance** with 60+ partners. Mandates (cryptographically-signed user intent) are the new pattern. Agorio should ship an initial `Ap2Client`.
- **WooCommerce** is the obvious second real-merchant adapter after Shopify.

v0.5 ships these three protocol updates *and* relicenses the plugins. The Pro tier is repositioned as "Agorio Cloud early access" ‚Äî Cloud itself ships in v0.6.

## Goals

1. Open-source all 5 governance plugins (MIT, on npm)
2. Handle Shopify's May 30 UCP migration without regressions
3. Ship initial AP2 client (behind feature flag)
4. Ship WooCommerce adapter
5. Reposition `/pricing` for Cloud early access

## Out of scope (deferred to later versions)

- Building Cloud itself (v0.6)
- Procurement reference agent (v0.7)
- BigCommerce adapter, security audit, EU AI Act exports (v0.8)
- Multi-agent orchestration, persistent sessions (v0.7)

### Step 3 ‚Äî AP2 client (initial, behind feature flag) (~5 days)

**Background:** AP2 (Agent Payments Protocol) is now FIDO-stewarded. Core concept: **Mandates** ‚Äî cryptographically-signed digital contracts that prove user intent. Two types:
- **Intent Mandate** ‚Äî "Buy concert tickets when they go on sale, up to $200"
- **Cart Mandate** ‚Äî signed after user approves a specific cart

**Files to create:**
- `src/client/ap2-client.ts` ‚Äî `Ap2Client` class with:
  - `createIntentMandate(intent: IntentMandateInput): Promise<SignedMandate>`
  - `createCartMandate(cart: CartMandateInput, intentMandate: SignedMandate): Promise<SignedMandate>`
  - `submitPayment(cartMandate: SignedMandate): Promise<Ap2PaymentResult>`
- `src/types/index.ts` ‚Äî add `IntentMandate`, `CartMandate`, `SignedMandate`, `Ap2PaymentResult` types
- `tests/ap2-client.test.ts` ‚Äî unit tests for mandate creation, signing, verification
- `src/index.ts` ‚Äî export `Ap2Client` and types

**Feature flag:**
- `AgentOptions.experimental_ap2?: boolean` ‚Äî defaults `false` for v0.5. Enabling routes payments through AP2 instead of ACP/UCP-native flows.

**Verification:**
- `tests/ap2-client.test.ts` passes with mocked FIDO signing
- README mentions AP2 as experimental

## Risks

- **Shopify UCP profile shape unknown until tested.** Mitigation: capture a real profile first via `curl` and write tests against the actual shape, not guesses.
- **AP2 spec may evolve before v0.5 ships.** Mitigation: ship as `experimental_ap2` feature flag; document instability in README.
- **WooCommerce REST API requires consumer key/secret pairs for write operations.** Read-only browsing works without auth; document this clearly in the adapter README.
